## Setup

```typescript
import { OneSwap } from '@oneswap/sdk'
const client = new OneSwap({ apiKey: 'os_live_...' })
```

| Option | Type | Default | Description |
|---|---|---|---|
| `apiKey` | `string` | required | Your API key |
| `timeout` | `number` | `30000` | Request timeout in ms |

## Pools

| Method | Description |
|---|---|
| `client.pools.list()` | All pools with effective reserves, fee, and LP supply |
| `client.pools.get(poolId)` | Pool detail with `swapAddress` and `lpAddress` |
| `client.pools.getStats(poolId, { days? })` | `apr`, `apy`, `volume24h`, `fees24h`, `lpFees24h`, `tvl`, `swapCount24h`, and APR metadata computed from fee events and effective reserves. Default lookback is 7 days. |

## Tokens

| Method | Description |
|---|---|
| `client.tokens.list()` | Available tokens with `name` and `admin` |

## Quotes

| Method | Description |
|---|---|
| `client.quotes.get({ from, to, amount, receiverParty? })` | Price quote. Includes `estimatedTrafficCost`, `trafficFeeInInput`, and `inputAmountAfterTrafficFee`. `outputAmount` is based on the net input after traffic recovery. |

## Wallets

| Method | Description |
|---|---|
| `client.wallets.create({ walletAddress })` | Register a direct Canton party for this developer |
| `client.wallets.get(walletAddress)` | Get a registered party |
| `client.wallets.list()` | List registered parties |
| `client.wallets.getBalances(walletAddress)` | Live token balances for a registered party |

## Swaps

| Method | Description |
|---|---|
| `client.swaps.create(params)` | Create swap intent |
| `client.swaps.getStatus(intentId)` | Poll swap status |
| `client.swaps.list(params?)` | List swaps |

### `swaps.create` params

| Param | Required | Description |
|---|---|---|
| `fromToken` | yes | Input token |
| `toToken` | yes | Output token |
| `amount` | yes | Input amount |
| `slippageTolerance` | no | `0.001`–`0.5`, default `0.01` |
| `walletAddress` | yes | User's actual Canton party ID |
| `outputAddress` | no | Deprecated. Ignored by the SDK client because the current execution flow always pays back to the depositor party. |

### SwapIntent object

Properties: `intentId`, `depositAddress`, `expectedOutput`, `minOutput`, `expiresAt`, `instructions`.

Notes:
- `depositAddress` is the pool swap party, not the user's own party.
- The deposit must be sent from the same `walletAddress` used to create the swap intent.
- A different wallet or Canton party will not complete the swap intent.
- `outputAddress` in the response is the resolved destination party. In the current flow it matches the depositor party.

| Method / Event | Description |
|---|---|
| `swap.wait(opts?)` | Await terminal state |
| `'processing'` | Deposit detected, swap executing |
| `'completed'` | Swap done |
| `'slippage_failed'` | Refunded because price moved |
| `'insufficient_amount'` | Refunded because input is below traffic cost |
| `'insufficient_liquidity'` | Refunded because pool liquidity is insufficient |
| `'output_failed'` | Output transfer failed and the input refund succeeded |
| `'no_output_holdings'` | No output inventory was available on the swap party |
| `'failed'` | Swap reached a generic terminal failure state |
| `'refund_failed'` | Refund path failed and manual intervention is required |
| `'expired'` | Timed out |

## Liquidity

| Method | Description |
|---|---|
| `client.liquidity.create(params)` | Create LP intent |
| `client.liquidity.getStatus(intentId)` | Poll LP status |
| `client.liquidity.list(params?)` | List LP intents |

### `liquidity.create` params

| Param | Required | Description |
|---|---|---|
| `poolId` | yes | Target pool |
| `amountA` | yes | Token A amount |
| `amountB` | yes | Token B amount |
| `walletAddress` | yes | User's actual Canton party ID |

### LpIntent object

Properties: `intentId`, `depositAddress`, `lpAddress`, `expiresAt`, `instructions`.

Notes:
- `depositAddress` / `lpAddress` is the pool LP party, not the user's own party.
- The user must send both deposits from the same `walletAddress`.
- A different wallet or Canton party will not complete the LP intent.
- LP intents expire after 30 minutes if the pair is not completed.
- LP positions are credited in OneSwap's database, not as a user-held on-ledger LP token.

| Method / Event | Description |
|---|---|
| `lp.wait(opts?)` | Await terminal state |
| `'partial'` | One token received |
| `'processing'` | Both tokens received and LP position is being credited |
| `'completed'` | LP position credited |
| `'refunded'` | Deposit was returned because the live pool no longer matched the intent |
| `'failed'` | LP settlement failed and needs investigation |
| `'expired'` | Timed out |
| `'cancelled'` | Intent was cancelled before completion |

## Positions

| Method | Description |
|---|---|
| `client.positions.list(walletAddress)` | LP positions with share percentage and current value based on effective reserves |
| `client.positions.getEarnings(walletAddress, poolId)` | APR, APY, projected earnings, and current withdrawable amounts using the same effective-reserve and fee-event APR calculation shown by the app |

## History

| Method | Description |
|---|---|
| `client.history.list(params?)` | Swap and LP intent history for your developer account |

## Track

| Method | Description |
|---|---|
| `client.track(partyId)` | All swaps and LP intents for a registered Canton party |

## Fee earnings

| Method | Description |
|---|---|
| `client.fees.getEarnings()` | Fee share config, payout mode, accumulated fees, total collected |
| `client.fees.getEarningsHistory(params?)` | Paginated fee events |
| `client.fees.getCollections(params?)` | Paginated fee collection history |
