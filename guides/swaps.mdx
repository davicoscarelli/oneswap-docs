title: Swaps
description: Execute token swaps with slippage protection and direct-party settlement.
---

## Basic swap

```typescript
const swap = await client.swaps.create({
  fromToken: 'Amulet',
  toToken: 'USDCx',
  amount: '100',
  walletAddress: 'alice::12205a8c...'
})
```

This returns a `SwapIntent` with everything you need:

| Field | Description |
|---|---|
| `depositAddress` | Pool swap party that should receive the deposit |
| `expectedOutput` | Estimated output amount |
| `minOutput` | Minimum accepted after slippage |
| `expiresAt` | Intent expiration timestamp |
| `instructions` | Step-by-step transfer instructions |

## Direct-party flow

1. Create the intent with the user's Canton party in `walletAddress`
2. OneSwap returns the pool swap party as `depositAddress`
3. The user sends the input token directly from that party to `depositAddress`
4. OneSwap detects the deposit and executes the swap
5. Output returns directly to the same party unless `outputAddress` is set

## Slippage tolerance

Default is 1%. You can set it between 0.1% (`0.001`) and 50% (`0.5`):

```typescript
const swap = await client.swaps.create({
  fromToken: 'Amulet',
  toToken: 'USDCx',
  amount: '100',
  slippageTolerance: 0.005,
  walletAddress: 'alice::12205a8c...'
})
```

If the actual output falls below `minOutput`, the swap is refunded.

## Send output to a different party

```typescript
const swap = await client.swaps.create({
  fromToken: 'Amulet',
  toToken: 'USDCx',
  amount: '100',
  walletAddress: 'alice::12205a8c...',
  outputAddress: 'some-other-party::namespace'
})
```

If you omit `outputAddress`, output is sent back to the depositor party.

## Track swap status

### Await completion

```typescript
const result = await swap.wait()
```

### Listen to events

```typescript
swap.on('processing', () => console.log('Deposit detected'))
swap.on('completed', (s) => console.log(`Got ${s.actualOutput}`))
swap.on('slippage_failed', () => console.log('Refunded: price moved'))
swap.on('insufficient_amount', () => console.log('Refunded: below traffic cost'))
swap.on('expired', () => console.log('Expired'))
```

### Poll manually

```typescript
const status = await client.swaps.getStatus(swap.intentId)
```

## Status lifecycle

```
pending → processing → sending_output → completed
                    → slippage_failed
                    → insufficient_amount
                    → insufficient_liquidity
                    → output_failed
         → expired
```

## Quotes

```typescript
const quote = await client.quotes.get({
  from: 'Amulet',
  to: 'USDCx',
  amount: '100'
})
```

`quote.estimatedTrafficCost`, `quote.trafficFeeInInput`, and `quote.inputAmountAfterTrafficFee` help you account for Canton execution costs before the swap executes.

- `trafficFeeInInput` is the amount deducted from the input token as traffic-cost recovery.
- `outputAmount` is quoted from `inputAmountAfterTrafficFee`, not from the gross deposit amount.
- The traffic fee is separate from the 2% swap fee and is not shared with LPs or SDK developers.
