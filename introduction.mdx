# OneSwap SDK

OneSwap lets you add token swaps and liquidity to any app on Canton Network.

```typescript
import { OneSwap } from '@oneswap/sdk'

const client = new OneSwap({ apiKey: 'os_live_...' })

// register the user's Canton party once
const wallet = await client.wallets.create({ walletAddress: 'alice::12205a8c...' })

// get a price quote
const quote = await client.quotes.get({
  from: 'Amulet',
  to: 'USDCx',
  amount: '100',
  receiverParty: wallet.walletAddress
})

// create a swap intent
const swap = await client.swaps.create({
  fromToken: 'Amulet',
  toToken: 'USDCx',
  amount: '100',
  walletAddress: 'alice::12205a8c...'
})

// wait for it to complete
const result = await swap.wait()
console.log(`Received ${result.actualOutput} USDCx`)
```

## What you get

- **Swap tokens** — Amulet (CC), USDCx, and other CIP-56 tokens
- **Direct-party flow** — users deposit from their own Canton party directly to pool parties
- **Real-time quotes** — prices, price impact, slippage protection, traffic-cost estimates
- **Liquidity provision** — add LP positions and track earnings
- **Event-driven** — poll or listen for swap completion

## How swaps work

1. You call `client.swaps.create()` with the user's Canton party ID
2. OneSwap returns the pool swap party as the `depositAddress`
3. The user sends tokens directly from that party to the pool party
4. OneSwap executes the swap automatically
5. Output returns directly to the same party that makes the deposit

That's it. No extra wallet layer, no backend forwarding step, no withdraw hop.
